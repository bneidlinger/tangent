üöÄ Project Idea

Simulate a stock market with 100+ distributed LLMs running as individual investor ‚Äúminds‚Äù across a Raspberry Pi mesh (or other distributed hardware). Each LLM node runs a 7B (or other) model, simulates a unique investor/trader persona, and makes buy/sell/hold decisions in response to real or synthetic market/news data.

  • Each LLM agent interprets news, prices, rumors, and acts according to its profile.
  • Market responses, bubbles, and panics emerge from agent interactions, not a single program.
  • Ultimate testbed for LLM-driven multi-agent simulation, decentralized inference, and market modeling.

üß† Core Architecture

  • 100x Pi 5 nodes (or split processes) running llama.cpp 7B models as individual investors
  • Orchestrator/hub (desktop, Pi, or cloud): sends market/news events, collects responses, manages round cycles
  • Profile Config: Each agent gets a persona file (risk, style, goals, biases, ‚Äúfears bear markets,‚Äù ‚Äúloves meme stocks,‚Äù etc.)
  • Real-time or simulated data feed (prices, news, tweets, earnings, etc.)
  • Aggregated outputs: market summary, order book, behavioral logs, etc.
# Example profile (YAML/JSON) - id: 23 type: "YOLO Daytrader" risk: "High" pattern: "Buys on hype, sells fast, tracks Twitter" - id: 47 type: "Index Dad" risk: "Low" pattern: "Buys dips, never sells, hates crypto"

üîÅ Simulation Flow

  1. Load profiles & LLMs on all nodes (each with a unique system prompt/context).
  2. At each timestep, the orchestrator broadcasts a news+market event to all LLMs.
  3. Each LLM generates a decision: BUY/SELL/HOLD; amount; reason
  4. Orchestrator aggregates responses, updates simulated price/order book based on agent actions.
  5. Next round: repeat with new data and new context (including past ‚Äúmarket‚Äù actions).
# Example prompt to agent: You are a "YOLO Daytrader" who follows meme stocks. Today's news: "TSLA misses earnings, CEO tweets 'we'll bounce back!'" Current price: $154.10 What do you do? Respond as: BUY/SELL/HOLD; amount; short reason.

üåê Scaling & Experimentation

  • Agents can be distributed across real Pi mesh (ideal) or simulated in processes/containers.
  • Coordinator can run locally or remotely, supports batch rounds or real-time streaming.
  • Profiles can be swapped, mutated, or assigned randomly for scenario stress tests.
  • Model size can be reduced to fit hardware constraints, or use mixed models per node for diversity.
