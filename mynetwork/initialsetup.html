Tangnet + AT&T All-Fi Hub Integration Guide

This guide outlines how to achieve secure, independent internet access for Tangnet nodes using AT&T Internet Air with the All-Fi Hub.

1. Hardware Overview

  • All-Fi Hub (combo modem/router)
  • Optional: Your own router/switch for custom networking
  • Tangnet Nodes: Raspberry Pi 5s, Windows machines, LED controllers

2. Setup Steps

  1. Unbox & Power On the All-Fi Hub. Let it boot and connect to the AT&T cell network.
  2. Verify Access: Connect a laptop to Wi-Fi or via Ethernet and ensure you can browse the internet.
  3. Access Gateway Settings: Navigate to http://192.168.1.254 and log in using the sticker password.
  4. Assign Static IPs or DHCP Reservations for your Pi mesh nodes.
  5. Port Forwarding: Set up any custom ports for remote access to nodes, SSH, etc.
  6. Connect Router/Switch: Plug your custom router or switch into an available LAN port if desired.

3. Tips

  • Use your router in Access Point mode to extend coverage without double NATing.
  • If needing full control, disable All-Fi Wi-Fi and route all traffic via your own router.
  • Use nmap or arp-scan on one node to find all connected devices quickly.
  • Add Tangnet aliases back into your ~/.bash_aliases after setting up new network.

4. Verification Checklist

  • ✅ Each Pi has an IP address and internet access
  • ✅ SSH working between all Tangnet nodes
  • ✅ Speed test from each device shows solid connection
  • ✅ Optional: Enable fail2ban and update all packages
